# Base Only Fix Report

## 1. What was broken

`Assets/Scenes/Base.unity` had a damaged UI hierarchy. The old Canvas object was missing, but several UI RectTransforms still referenced its local file identifier. Because of that, Unity reported broken PPtr errors while opening the scene and the Base UI could load as empty or partially missing.

`UnitIcon` also had a component entry pointing to a non-existent local component id. Unity removed that missing component on scene load, producing the `Component at index ... could not be loaded` / `Creating missing CanvasRenderer` style errors.

The old `THBaseRuntime` depended on serialized template references (`buildingCardTemplate`, `armyRowTemplate`, container references). Once the scene references were broken, it could no longer reliably draw hire rows, army rows, or the back button.

## 2. Broken references found

- `UnitIcon` referenced missing component id `1930271375`.
- Multiple UI objects referenced missing parent RectTransform id `1971326526`.

These local missing references were removed from `Base.unity`.

## 3. What was removed or rebuilt

- The broken `UnitIcon` missing component reference was removed.
- UI objects no longer point at the deleted Canvas RectTransform.
- `THBaseRuntime` was rewritten so Base can rebuild a clean Canvas at runtime if the serialized Canvas is missing or damaged.
- Added editor command `The Hero/Base/Fix Base Scene Only` in `Assets/Editor/TheHeroFixBaseOnly.cs` to delete the old Base Canvas and rebuild it cleanly.

## 4. New Base Canvas structure

The rebuilt Canvas uses:

- `Canvas`
- `TopBar`
  - `TitleText`
  - `ResourcesText`
  - `BackToMapButton`
- `MainPanel`
  - `HirePanel`
    - `HireTitle`
    - `HireListContainer`
  - `ArmyPanel`
    - `ArmyTitle`
    - `ArmyListContainer`
    - `ArmySummaryText`
- `MessagePanel`
  - `MessageText`

Panels are dark, decorative images do not block raycasts, and buttons remain raycastable.

## 5. Connected buttons

- `BackToMapButton` calls `THBaseRuntime.BackToMap()`.
- Hire rows are generated by `THBaseRuntime.RefreshUI()`.
- `RecruitOneButton` calls `RecruitOne(unitId)`.
- `RecruitAllButton` calls `RecruitAll(unitId)`.

## 6. Manual check

1. In Unity, run `The Hero/Base/Fix Base Scene Only`.
2. Open `Assets/Scenes/Base.unity`.
3. Press Play.
4. Confirm resources are visible.
5. Confirm hire rows show `Мечник`, `Лучник`, `Маг`.
6. Confirm `Маг x0` is not shown in the Army panel.
7. Press `Нанять 1` and confirm resources and army update.
8. Press `Всех` and confirm availability updates.
9. Press `НА КАРТУ` and confirm the Map scene loads.
10. Confirm Console has no red Base scene errors.
